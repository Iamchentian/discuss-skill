# Discuss Skill

一个完全自包含的结构化议题研讨技能，基于金字塔原理（SCQA 框架），支持本地、对话、飞书云端三种持久化方式。

## 特性

- **核心功能内置**：议题拆解（MECE）、深度追问、verdict 裁决均已内置，无需依赖其他技能
- **三种持久化模式**：本地文件 / 纯对话 / 飞书云端，按会话能力自动检测
- **verdict 质量裁决**：每个议题追问后给出 solid / needs-work / invalid 评级，强制对论据质量负责
- **硬性拆解上限**：递归深度 ≤4 层，子议题 ≤5 个，避免无限递归和信息过载
- **异常处理**：覆盖非法输入、云端失败、结论推翻、上下文超长等边界场景
- **SCQA 文档**：结构化的背景-冲突-问题-解决方案文档，含落地规划与风险应对

## 适用场景

- 复杂问题需要结构化分析
- 团队讨论和决策制定
- 战略规划和方案设计
- 需要明确执行步骤和时间节点

## 快速开始

### 1. 安装

将 `SKILL.md` 及 `references/`、`assets/` 目录复制到你的智能体技能目录中。

### 2. 触发

使用以下任一触发词：
- `讨论` / `讨论下`
- `/discuss`
- `discuss this`

### 3. 使用示例

```
用户：/discuss 分析为什么最近项目延期严重

技能会自动：
1. 检测持久化模式（本地/对话/云端）
2. 按 MECE 拆解议题为 3-5 个子议题
3. 引导逐个讨论，深度追问后给出 verdict
4. 循环直至所有议题 ✅
5. 生成 SCQA 文档 + 落地规划
```

## 流程步骤

```
1. 定义根问题（必要时追问 1-2 个澄清问题）
2. [可选] 补充上下文（仅根问题信息不足时才追问背景）
3. 检测持久化模式
4. 拆解议题（MECE，3-5 个），持久化
5. 展示清单，用户选择议题
6. 讨论议题：递归拆解 或 深度追问 → verdict
7. 循环 5-6 直至所有叶子议题 ✅
8. SCQA 汇总 + 落地规划，持久化
```

**上下文追问是可选的**——默认直接进入拆解，不要为每个议题强制问"为什么讨论这个"。

## 持久化模式

按优先级检测，用户显式指定时以用户为准。未指定时默认**对话模式**（安全兜底）。

| 模式 | 触发方式 | 检测条件 |
|------|----------|----------|
| **本地模式** | `--本地` / `--local` | 或会话可用 write/edit 工具 |
| **对话模式** | `--对话` / `--chat` | 默认兜底 |
| **云端模式** | `--云端 <飞书链接>` | 需 feishu connector 已连接 |

### 本地模式
文件保存在 `docs/discuss/YYYY-MM-DD-[主题]/` 下，含 notes.md 和 summary.md。

### 对话模式
纯对话展示，不进行文件操作。用户需手动复制保存。

### 云端模式（飞书）
用户提供的飞书文档链接作为父文档，通过 `lark-doc` skill 在其下创建子文档：
- `<议题名>-讨论过程`（对应 notes）
- `<议题名>-SCQA`（对应 summary）

> 语雀等其他云文档平台当前无官方 connector 支持，建议改用本地模式后手动迁移。

## Verdict 裁决机制

深度追问后必须给出 verdict：

| verdict | 含义 | 处理 |
|---------|------|------|
| `solid` | 论证充分 | 标记 ✅ |
| `needs-work` | 论据有漏洞 | 补充追问 1-2 轮 |
| `invalid` | 前提不成立 | 回退拆解或调整根问题 |

详细示例见 `references/verdict-examples.md`。

## SCQA 框架

| 元素 | 说明 | 输出要求 |
|------|------|----------|
| **S (Situation)** | 当前背景和现状 | 客观描述，用数据支撑 |
| **C (Complication)** | 遇到的冲突和挑战 | 明确痛点和障碍 |
| **Q (Question)** | 核心要解决的问题 | 清晰、可回答的问题 |
| **A (Answer)** | 解决方案 | 可执行的行动步骤 |

完整文档结构（含议题分析汇总、落地规划、风险与应对）见 `assets/summary-template.md`。

## 拆解规则

- **MECE 检查**：子议题相互独立、完全穷尽
- **子议题数量**：3-5 个，最多不超过 5 个
- **递归深度上限**：4 层，超过必须转入深度追问
- **verdict 必填**：禁止跳过 verdict 直接标记 ✅
