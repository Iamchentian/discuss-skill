# Discuss Skill

一个完全自包含的结构化议题研讨技能，基于金字塔原理（SCQA框架），支持多种持久化方式。

## 特性

- ✅ **无外部依赖**：所有功能内置，无需安装其他技能
- ✅ **多平台持久化**：支持本地文件、语雀、飞书、纯对话模式
- ✅ **自动检测**：根据环境自动选择可用的持久化方式
- ✅ **降级方案**：任一方式不可用时自动降级
- ✅ **SCQA 框架**：结构化的问题分析和解决方案文档

## 适用场景

- 复杂问题需要结构化分析
- 需要生成完整的解决方案文档
- 团队讨论和决策制定
- 战略规划和方案设计
- 需要明确执行步骤和时间节点

## 快速开始

### 1. 安装

将 `SKILL.md` 文件复制到你的智能体技能目录中。

### 2. 触发

使用以下任一触发词：
- `讨论`
- `讨论下`
- `分析这个话题`
- `/discuss`
- `analyze`

### 3. 使用示例

```
用户：/discuss 分析为什么最近项目延期严重

技能会自动：
1. 检测可用的持久化方式
2. 拆解议题为子议题
3. 引导你逐个讨论
4. 生成 SCQA 文档
```

## 持久化模式

### 本地文件模式（推荐）

适用于本地智能体，文件保存在 `docs/discuss/` 目录下。

```
docs/discuss/
└── YYYY-MM-DD-[主题名称]/
    ├── notes.md      # 讨论过程记录
    └── summary.md    # 最终汇总文档
```

### 语雀模式

适用于云端智能体，文档保存在语雀空间中。

### 飞书模式

适用于云端智能体，文档保存在飞书文档中。

### 纯对话模式

适用于无文件能力的智能体，内容仅在对话中展示。

## SCQA 框架

| 元素 | 说明 | 输出要求 |
|------|------|----------|
| **S (Situation)** | 当前背景和现状 | 客观描述，用数据支撑 |
| **C (Complication)** | 遇到的冲突和挑战 | 明确痛点和障碍 |
| **Q (Question)** | 核心要解决的问题 | 清晰、可回答的问题 |
| **A (Answer)** | 解决方案 | 可执行的行动步骤 |
