# Discuss Skill

一个完全自包含的结构化议题研讨技能，基于金字塔原理（SCQA框架），支持多种持久化方式。

## 特性

- ✅ **核心功能内置**：议题拆解和深度追问已内置，无需依赖其他技能
- ✅ **多平台持久化**：支持本地模式、对话模式、云端文档模式
- ✅ **灵活指定**：用户可手动指定模式，未指定时默认对话模式
- ✅ **SCQA 框架**：结构化的问题分析和解决方案文档

## 适用场景

- 复杂问题需要结构化分析
- 需要生成完整的解决方案文档
- 团队讨论和决策制定
- 战略规划和方案设计
- 需要明确执行步骤和时间节点

## 快速开始

### 1. 安装

将 `SKILL.md` 文件复制到你的智能体技能目录中。

### 2. 触发

使用以下任一触发词：
- `讨论`
- `讨论下`
- `分析这个话题`
- `/discuss`
- `analyze`

### 3. 使用示例

```
用户：/discuss 分析为什么最近项目延期严重

技能会自动：
1. 询问上下文：为什么要讨论这个话题
2. 定义问题：明确根问题
3. 拆解议题：生成子议题
4. 引导逐个讨论
5. 生成 SCQA 文档
```

## 流程步骤

```
1. 补充上下文：用户说明为什么要讨论这个话题（背景、动机、触发事件）
2. 定义问题：明确根问题
3. 拆解议题：内置拆解，生成子议题
4. 展示清单：列出所有子议题及状态
5. 用户选择：选择要讨论的议题
6. 讨论议题：递归拆解或深度追问
7. 循环直至所有议题解决
8. SCQA 汇总：生成文档
9. 落地规划：制定执行节奏
```

## 持久化模式

### 三种模式

| 模式 | 触发词 | 适用场景 |
|------|--------|----------|
| **本地模式** | `本地` / `local` / `file` | 本地智能体，文件保存在本地 |
| **对话模式** | `对话` / `chat` / `pure`（默认） | 无文件能力的智能体，纯对话 |
| **云端文档模式** | `云端` / `cloud`（需提供文档地址） | 云端智能体，保存到用户指定的云文档 |

**用户未指定时，默认使用对话模式。**

### 本地模式

文件保存在本地，支持 mkdir/write/edit。

### 对话模式

纯对话，用户需手动复制保存。

### 云端文档模式

用户提供云文档地址（如：飞书文档链接、语雀知识库链接等），技能自动在该地址下创建 notes 和 summary。

## SCQA 框架

| 元素 | 说明 | 输出要求 |
|------|------|----------|
| **S (Situation)** | 当前背景和现状 | 客观描述，用数据支撑 |
| **C (Complication)** | 遇到的冲突和挑战 | 明确痛点和障碍 |
| **Q (Question)** | 核心要解决的问题 | 清晰、可回答的问题 |
| **A (Answer)** | 解决方案 | 可执行的行动步骤 |
