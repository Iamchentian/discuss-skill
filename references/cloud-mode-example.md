# 云端模式示例（飞书）

## 场景

分析为什么最近项目延期严重，用户指定将讨论内容保存到飞书文档。

## 前置条件

- `feishu` connector 已连接
- 用户提供了一个飞书文档链接（作为父文档，在其下创建子文档）

## 初始化

```
用户：/discuss 分析为什么最近项目延期严重 --云端 https://xxx.feishu.cn/docs/parentDocToken

技能：检测到飞书 connector 已连接，使用云端模式
技能：从链接中提取文档 token：parentDocToken
技能：调用 lark-doc skill，在父文档下创建子文档：
  - 过程文档：项目延期分析-讨论过程
  - 汇总文档：项目延期分析-SCQA
技能：文档创建成功！
  - 讨论过程：https://xxx.feishu.cn/docs/notesToken
  - SCQA 汇总：https://xxx.feishu.cn/docs/summaryToken
```

## 对话流程

```
技能：根问题：项目延期问题分析

技能：拆解议题（MECE 检查通过）：
├─ 1. 资源问题
├─ 2. 流程问题
└─ 3. 技术问题

技能：调用 lark-doc skill 更新"项目延期分析-讨论过程"，写入议题清单

技能：
┌─────────────────────────────────────────────────────────────┐
│ 📋 议题清单                                                 │
├─────────────────────────────────────────────────────────────┤
│ ❌ 1. 资源问题                                              │
│ ❌ 2. 流程问题                                              │
│ ❌ 3. 技术问题                                              │
├─────────────────────────────────────────────────────────────┤
│ 进度：0/3 议题已解决                                         │
└─────────────────────────────────────────────────────────────┘

请选择要讨论的议题：1

用户：1

技能：用户选择：1. 资源问题
技能：调用 lark-doc skill 更新讨论过程文档，议题 1 状态 → ⏳

💾 持久化状态：内容已保存至飞书文档 [项目延期分析-讨论过程](https://xxx.feishu.cn/docs/notesToken)
```

## 异常处理示例

```
场景：feishu connector 未连接

用户：/discuss 分析项目延期 --云端 https://xxx.feishu.cn/docs/xxx

技能：检测到飞书文档链接，但 feishu connector 未连接。
技能：请先在连接器管理中启用飞书 connector，或选择以下方式：
  1. 等你启用飞书后继续云端模式
  2. 回退到本地模式（保存到本地文件）
  3. 回退到对话模式（不持久化）
```

## 飞书文档结构

### 过程文档：项目延期分析-讨论过程

与 `assets/notes-template.md` 结构一致，包含议题清单表、讨论详情、中断记录。

### 汇总文档：项目延期分析-SCQA

与 `assets/summary-template.md` 结构一致，包含 SCQA 四段 + 议题分析汇总 + 落地规划 + 风险与应对。

## 注意事项

- 语雀等其他云文档平台当前无官方 connector 支持。若用户提供语雀链接，提示改用本地模式后手动迁移，或使用对话模式
- 每次 lark-doc 更新失败时，重试 1 次；仍失败则询问用户是否回退本地模式
- 避免高频更新飞书文档（API 限流），状态变更批量合并后再调用 lark-doc
