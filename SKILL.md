# Discuss — 结构化议题研讨

基于金字塔原理（Barbara Minto），将复杂议题逐层拆解为子议题，通过深度追问和脑暴交替推进，最终汇总为完整 SCQA 文档与落地规划。

**核心功能内置**：议题拆解和深度追问已内置，无需依赖其他技能。可自由调用 websearch、文件操作、云文档 API 等工具。

**版本**：v1.1.0 | **更新日期**：2025-06-27

## When to Use

**触发词**：讨论、讨论下、分析这个话题、/discuss、analyze

**适用场景**：
- 复杂问题需要结构化分析
- 需要生成完整的解决方案文档
- 团队讨论和决策制定
- 战略规划和方案设计
- 需要明确执行步骤和时间节点

## Quick Start

```
用户：/discuss 分析为什么最近项目延期严重

技能自动：
1. 检测持久化模式（本地/对话/云端文档）
2. 拆解议题为子议题
3. 引导逐个讨论
4. 生成 SCQA 文档
```

## Core Pattern

### 递归拆解流程

```
根问题 → 拆解为N个子议题 → 每个子议题：可继续拆解？→ 是 → 递归拆解
                                              └→ 否 → 深度追问 → 获得论据
所有议题解决 → 汇总成完整SCQA文档
```

### 判断标准

**可继续拆解**：议题复杂、多维度、需进一步明确细节

**进入深度追问**：议题具体、需深挖根因、需验证假设

## 议题拆解（内置）

**步骤**：
1. 理解根问题：明确核心问题
2. 识别维度：从不同角度分析（原因、影响、解决方案）
3. MECE 检查：确保子议题相互独立、完全穷尽
4. 生成子议题：通常 3-5 个
5. 标记状态：初始为 ❌

**示例**：
```
根问题：为什么最近项目延期严重？
├─ 1. 资源问题（人员不足、技能不匹配）
├─ 2. 流程问题（需求变更频繁、评审环节多）
└─ 3. 技术问题（技术债务）
```

## 深度追问（内置）

**步骤**：
1. 无情盘问：对每个假设、前提、数据依赖逐一质疑
2. 逐分支遍历：逻辑前提、数据来源、因果关系、替代解释、边界条件、实施可行性
3. 逐个决策：每个质疑点给出推荐答案
4. 给出 verdict：
   - `solid`：论证充分，逻辑自洽
   - `needs-work`：有漏洞，需要修正
   - `invalid`：前提不成立，方向错误

**示例**：
```
议题：人员不足
质疑：当前团队规模8人，项目需求12人
verdict：solid（数据支撑充分）
```

## 流程步骤

```
1. 补充上下文：用户说明为什么要讨论这个话题（背景、动机、触发事件）
2. 定义问题：明确根问题
3. 拆解议题：内置拆解，生成子议题，【持久化】保存
4. 展示清单：列出所有子议题及状态
5. 用户选择：选择要讨论的议题，【持久化】更新状态为 ⏳
6. 讨论议题：递归拆解或深度追问，【持久化】记录论据
7. 循环直至所有议题解决
8. SCQA 汇总：生成文档，【持久化】保存
9. 落地规划：制定执行节奏，【持久化】更新
```

**⚠️ 关键：每个【持久化】步骤都必须执行对应的文件操作！**
### 三种模式

| 模式 | 触发词 | 适用场景 |
|------|--------|----------|
| **本地模式** | `本地` / `local` / `file` | 本地智能体，文件保存在本地 |
| **对话模式** | `对话` / `chat` / `pure`（默认） | 无文件能力的智能体，纯对话 |
| **云端文档模式** | `云端` / `cloud`（需提供文档地址） | 云端智能体，保存到用户指定的云文档 |

**用户未指定时，默认使用对话模式。**

### 本地模式

**文件结构**：
```
docs/discuss/
└── YYYY-MM-DD-[主题名称]/
    ├── notes.md      # 讨论过程记录
    └── summary.md    # 最终汇总文档
```

**操作命令**：
- 创建目录：`mkdir -p docs/discuss/YYYY-MM-DD-[主题]/`
- 创建文件：`write docs/discuss/YYYY-MM-DD-[主题]/notes.md`
- 更新文件：`edit docs/discuss/YYYY-MM-DD-[主题]/notes.md`

**状态展示**：`💾 持久化状态：内容已保存至 docs/discuss/YYYY-MM-DD-[主题]/notes.md`

### 对话模式

**特点**：
- 不进行任何文件操作
- 所有内容仅在对话中展示
- 用户需要手动复制保存

**状态展示**：`💾 持久化状态：对话模式，内容仅在会话内有效`

### 云端文档模式

**工作流程**：
1. 用户提供云文档地址（如：飞书文档链接、语雀知识库链接等）
2. 技能在该地址下创建过程文档：`<议题名>-讨论过程`
3. 创建汇总文档：`<议题名>-SCQA`

**状态展示**：`💾 持久化状态：内容已保存至云文档 [<文档名>](<url>)`

**使用示例**：
```
用户：/discuss 分析项目延期问题 --云端 https://xxx.feishu.cn/docs/xxx

技能：正在在指定地址下创建文档...
- 创建过程文档：项目延期分析-讨论过程
- 创建汇总文档：项目延期分析-SCQA

技能：文档创建成功！开始讨论...
```
| ✅ | 已解决 |

### SCQA 文档

```markdown
# [主题名称] SCQA分析

## 背景 (Situation)
[客观描述现状，用数据支撑]

## 冲突 (Complication)
[明确痛点和障碍]

## 问题 (Question)
[核心要解决的问题]

## 解决方案 (Answer)
[可执行的行动步骤]

## 落地规划
[执行步骤、执行节奏、风险与应对]
```

## 注意事项

- **递归深度**：避免过度拆解，一般3-4层为宜
- **议题独立性**：子议题之间尽量减少重叠
- **论据充分性**：深度追问要获得足够论据支持
- **状态跟踪**：实时更新每个议题的解决状态
- **及时汇总**：所有议题解决后立即汇总文档
- **灵活导航**：允许用户随时切换议题，支持中断和恢复
- **强制保存**：每次状态变更必须立即持久化，防止进度丢失

## 议题导航

### 用户操作

| 操作 | 说明 |
|------|------|
| 输入序号 | 讨论指定议题（如：1、2.1） |
| `切换到 X` | 中断当前议题，切换到议题X |
| `查看 X` | 查看议题X的详情 |
| `返回清单` | 返回议题清单 |
| `结束讨论` | 结束讨论，汇总文档 |

### 中断处理

1. **立即保存**当前讨论进度
2. 标记议题状态为进行中（⏳）
3. 切换到用户指定的议题

**强制保存规则**：
- 任何导致议题状态变更的操作都必须先保存
- 切换议题前必须保存当前议题的进度
- 退出 discuss 前必须保存所有未保存的进度

### 状态流转

```
❌未讨论 → ⏳进行中 → ✅已解决
            ↑    │
            └────┘ (中断后恢复)
```

## SCQA 框架

| 元素 | 说明 | 输出要求 |
|------|------|----------|
| **S (Situation)** | 当前背景和现状 | 客观描述，用数据支撑 |
| **C (Complication)** | 遇到的冲突和挑战 | 明确痛点和障碍 |
| **Q (Question)** | 核心要解决的问题 | 清晰、可回答的问题 |
| **A (Answer)** | 解决方案 | 可执行的行动步骤 |
